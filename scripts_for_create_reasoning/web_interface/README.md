# Веб-интерфейс системы оценки рисков

Веб-интерфейс на основе **Streamlit** для системы оценки рисков безопасности веб-приложений.

## Быстрый старт

### 1. Установка зависимостей

```bash
cd /ontology-for-assessing-the-information-security-risk-of-applications
source .venv/bin/activate
pip install -r requirements.txt
```

### 3. Запуск приложения

```bash
cd scripts_for_create_reasoning/web_interface
streamlit run app.py
```

После запуска в браузере откроется страница по адресу `http://localhost:8501`

## Возможности интерфейса

### Этапы анализа

1. **Ввод компонентов** — ввод названий и версий компонентов (например: `python 3.9, google chrome 90.0`)
2. **Выбор версий** — выбор точной версии из базы или ручной ввод
3. **Оценка важности** — установка коэффициента важности для каждого компонента
4. **Результаты** — таблица с рисками по компонентам и общий риск приложения
5. **Детали анализа** — подробная информация по уязвимостям (CVE) для каждого компонента

### Отображение результатов

- **Итоговый риск** — цветной блок с уровнем риска приложения
- **Таблица компонентов** — сводка по каждому компоненту (CVE, макс./сред. риск)
- **Детальный расчёт** — информация по каждой уязвимости (CVSS, EPSS, итоговый риск)

### Экспорт данных

- **CSV** — выгрузка результатов в табличном формате

## Шкала рисков

| Уровень | Диапазон | Цвет |
|---------|----------|------|
| Низкий | 0–5 | 🟢 Зелёный |
| Средний | 6–11 | 🟡 Жёлтый |
| Высокий | 12–17 | 🟠 Оранжевый |
| Критический | 18–24 | 🔴 Красный |

## Коэффициенты важности компонентов

| Значение | Уровень | Коэффициент |
|----------|---------|-------------|
| 1 | Low | 0.25 |
| 2 | Medium | 0.50 |
| 3 | High | 0.75 |
| 4 | Critical | 1.00 |

## Источники данных

- **NVD** — CVE (уязвимости)
- **MITRE CWE** — слабости
- **MITRE CAPEC** — шаблоны атак
- **FIRST EPSS** — вероятность эксплуатации

## Требования

- Python 3.8+
- Streamlit 1.28+
- Owlready2 0.40+
- Доступ к API FIRST EPSS (для получения вероятности эксплуатации)
